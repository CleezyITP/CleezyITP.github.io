JavaScript Review

Introduction

Javascript is used to make websites interactive. Javascript is a set of rules/instructions for your browser to follow. These step by step instructions are like directions or a recipe.

It is used to access and modify content in your webpages. You can use javascript to select any element, attibute or text in your HTML and either change them in some way or add new elements, attibutes or text.

Secondly, you can use javaScript to program a series of actions, or rules involved in changing your HTML. For example, you could program a currency calculator to change Ugandan shillings to US dollars, or convert the temperature from Farenheit to celsius and display the results on the screen.

Finally, you can create scripts that react to certain events in the browser, like when a button is pressed, a link is selected, or a certain amount of time has passed.

computers create "models" using data. These models are comprised of objects. Objects contain:

  • properties that tell us some information about the object
  • methods that that perform tasks based on properties
  • events that are triggered when users interact with the computer
To put this into context, the web browser you use (firefox, chrome, etc.) uses HTML markup to create a model of a webpage. Each element creates its own object/node that has attribute properties (classes, ids, alt text, etc.). Javascript makes webpages interactive by attaching events to the browser's model of a webpage

Where to include Javascript

Like CSS, you can add javascript directly into the body of your HTML files using a script tag. For example, 

<html>
<head>
</head>
<body>
<h1>my heading<h1>
<script>some code</script>
</body>
</html>
However, it is always best to create javascript in a seperate file and save it with a dot js file type. For example, I could name my file script.js.

How to write JavaScript...

In order to write javascript, you must first define your goal, then break each step down into smaller steps. The most effective way to write code, is to first write pseudo-code, or to write down your goal and the series of tasks that are required to complete it, in human language.
